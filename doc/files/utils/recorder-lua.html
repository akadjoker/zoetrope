Recorder

This records the user’s inputs for playback at a later time, or saving to a file.  You must start recording after all input objects are set up, and you should only run one recorder at a time.

The recorder is currently playing back user input.

Properties

mousePosInterval

How often, in seconds, we should capture the mouse position.  This records the mouse position when other events occur, too, so it’s possible the interval will be even higher in reality.  Setting this number lower will burn memory.

state

One of the state constants, indicates what the recorder is currently doing.

record

A table of inputs with timing information.

Functions

startRecording

Begins recording user inputs.  If the recorder is already recording, this has no effect.

Arguments

recordRecord to use.  Any existing data is appended to.  If omitted, the current record is used.  If the current record is unset, this creates a new record.

Returns

nothing

stopRecording

Stops recording user inputs.  If the recorder wasn’t recording anyway, this does nothing.

Arguments

none

Returns

nothing

startPlaying

Starts playing back user inputs.  If this is already playing a recording, this restarts it.

Arguments

recordRecord to play back.  If ommitted, this uses the recorder’s record property.

Returns

nothing

stopPlaying

Stops playing back user inputs.

Arguments

none

Returns

nothing
