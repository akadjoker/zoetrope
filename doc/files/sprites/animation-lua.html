Events

onEndSequence

Called whenever an animation sequence ends.  It is passed the name of the sequence that just ended.

Extends

Sprite

Properties

paused

Set this to true to freeze the animation on the current frame.

sequences

A lookup table of sequences.  Each one is stored by name and has the following properties:

  • frames - table of frames to display.  The first frame in the sheet is at index 1.
  • fps - frames per second.
  • loops - does the animation loop? defaults to true

currentSequence

A reference to the current animation sequence table.

currentName

The name of the current animation sequence.

currentFrame

The current frame being displayed; starts at 1.

frameIndex

Numeric index of the current frame in the current sequence; starts at 1.

frameTimer

Time left before the animation changes to the next frame in seconds.  Normally you shouldn’t need to change this directly.

Functions

play

Begins playing an animation in the sprite’s library.  If the animation is already playing, this has no effect.

Arguments

namename of the animation

Returns

nothing

freeze

Freezes the animation on the specified frame.

Arguments

  • index - integer frame index relative to the entire sprite sheet, starts at 1.  If omitted, this freezes the current frame.  If there is no current frame, this freezes on the first frame.

Returns

nothing
