Tweener

A tweener transitions a property from one state to another in in-game time.  A tweener instance is designed to manage many of these transitions at once, in fact.  In order for it to work properly, it must receive update events, so it must be added somewhere in the current view or app.  If you are using the View class, this is already done for you.

Properties

easers

These are different methods of easing a tween, and can be set via the ease property of an individual tween.  See http://www.gizma.com/easing/ for details.

Functions

reverse

A utility function; if set as an onComplete handler for an individual tween, it reverses the tween that just happened.  Use this to get a tween to repeat back and forth indefinitely (e.g. to have something glow).

reverseOnce

A utility function; if set as as an onComplete handler for an individual tween, it reverses the tween that just happened-- then stops the tween after that.

start

Begins a tweened transition.  All arguments are passed via properties of a single object as follows:

Arguments

targettarget object
propertyname of property of the target object to tween; can be either a number or a table of numbers (e.g. a color)
destinationdestination value, either number or color table
gettergetter function for the property; overrides property
settersetter function for the property; overrides property
durationhow long the tween should last in seconds, default 1
forceoverride any pre-existing tweens on this object/property?
easefunction name (in Tweener.easers) to use to control how the value changes
onCompletefunction to call when the tween finishes; is passed the individual tween object

Returns

nothing

stop

Stops a tween.

Arguments

targettween target
propertyname of property being tweened; if omitted, stops all tweens on the target

Returns

nothing
