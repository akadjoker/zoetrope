BitmapFont

This represents a proportional bitmap font, where each letter may be a different width.  The source image has to be formatted in a specific manner, with a sentinel row above a run of glyphs.  Each glyph is separated by a pixel in the columns to the left and the right of the sentinel row above.  The color of these pixels must be the exact color at (0, 0) in the image.  A source image can contain any number of sentinel row/glyph row pairs.

An ASCII art demonstration of a tiny capital A and C

x.....x...x  <--- sentinel row
...x...xxx.  <--- glyph data
..x.x..x...
.xxxxx.x...
.x...x.xxx.

Any extra glyph data after the last sentinel pixel is ignored.

Once created, a BitmapFont doesn’t have much in the way of user-serviceable parts.  Connect it to a <BitmapText> object to display some text onscreen.

This class is based heavily on FlxBitmapFont by Brandon Cashhttp://www.brandoncash.net/.

Properties

image

The source image to use.  Changing this property after creating an instance has no effect.

height

The height of a single row of glyphs in the source image, in pixels.  Every glyph must have the same height.  Changing this property after creating an instance has no effect.

alphabet

A string listing the order of characters’ appearance in the source image.  By default, it uses ASCII order.
